The paper page is the main research object on this site. It names the question,
states the current claim, gives the method, and links the surrounding material:
code, traces, models, measurements, and technical notes.

Notes have a smaller job. A note can record a measurement choice, a protocol
edge case, a modeling decision, or a useful implementation detail. It does not
need the full structure of a paper.

Technical material stays attached to the paper that gives it context.

| Surface | Job | Typical contents |
| --- | --- | --- |
| Paper | Present the research record | question, method, evidence, scope |
| Note | Explain a technical choice | measurements, sketches, examples |
| Material | Show the implementation surface | code, commands, traces, models |

## Page shape

A paper page should answer the next technical question without making the reader
guess where the evidence lives.

```yaml
paper:
  question: "What behavior is being measured?"
  method:
    - trace collection
    - normalized reporting
    - certificate-chain inspection
  supporting_material:
    - code
    - measurements
    - model notes
  scope:
    - sampling assumptions
    - parser assumptions
    - deployment drift
```

The structure keeps claim, method, evidence, and scope visible on the same path.

```ts
type PaperRecord = {
  claim: string;
  method: string[];
  evidence: Array<"code" | "trace" | "model" | "measurement" | "review">;
  scope: string[];
};
```

## Reading path

The reader should be able to start from a paper, open the explanation, and then
move to code or notes only when they want detail. That is the whole navigation
model.
