A technical portfolio works best when the reading path is obvious.

The homepage gives orientation. Papers carry the main record. Writing explains
technical choices. Code and measurements stay attached to the paper that uses them.
The about page gives the academic frame.

## Useful constraints

- Fast pages
- Visible categories
- Paper pages with question, method, evidence, and scope
- Visual structure tied to reading and comparison
- Technical detail as the main status signal

That structure keeps the work inspectable from the first page.
