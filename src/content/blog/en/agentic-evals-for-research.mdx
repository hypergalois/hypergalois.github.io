Agentic systems are easy to demo and hard to evaluate. A good run looks fluent;
a bad run may still look fluent unless the trace is inspectable.

The evaluation object should include the task, tools, intermediate decisions,
files touched, failures, recovery behavior, and final evidence. Without that, the
comparison is mostly vibes.

## A better harness

- Scenario inputs
- Tool-call trace
- File diffs and artifacts
- Failure labels
- Human review notes

This is not bureaucracy. It is how automated work becomes debuggable.
