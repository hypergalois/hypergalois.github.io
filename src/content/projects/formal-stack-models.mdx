## Problem

Security arguments often depend on small state transitions that are easy to
describe informally and hard to preserve under implementation pressure.

## Artifact

This line of work explores compact formal models that make the state, operations,
and invariants explicit enough to be reviewed and extended.

## Use

Formalization turns broad security statements into smaller objects: state,
transitions, invariants, and proof obligations.
